Quby : Conventions draft

R function arguments

R accepts arguments in functions that start with a dot ., for example:

1  # bad
2  hmac <- function(.arg1, .arg2, ...) {
3    statements
4    return(object)
5  }

While itâ€™s a correct way in R to define variables and arguments, we prefer a variable that can be mapped to a JSON and a Ruby. Variables starting with a dot will result in an error. Therefore the following function definition is better:

1  # good, note the arguments!
2  hmac <- function(arg1, arg2, ...) {
3    statements
4    return(object)
5  }
